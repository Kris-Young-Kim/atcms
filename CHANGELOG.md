## 형식 설명

### 버전 형식

각 버전은 `[MAJOR.MINOR.PATCH]` 형식을 따릅니다:

- **MAJOR**: 호환되지 않는 API 변경
- **MINOR**: 하위 호환성을 유지하면서 기능 추가
- **PATCH**: 하위 호환성을 유지하면서 버그 수정

### 변경 유형

- **추가**: 새로운 기능 추가
- **변경**: 기존 기능 변경
- **폐기**: 곧 제거될 기능
- **제거**: 제거된 기능
- **수정**: 버그 수정
- **보안**: 보안 취약점 수정

### 예시

```markdown
## [1.2.0] - 2025-12-01

### 추가
- 새로운 대시보드 차트 컴포넌트
- PDF 내보내기 기능

### 변경
- 대상자 검색 성능 개선
- UI 디자인 업데이트

### 수정
- 대여 반납 시 수량 계산 오류 수정
- 파일 업로드 크기 제한 오류 수정

### 보안
- SQL 인젝션 취약점 수정
```

## 작성 가이드

### 커밋 시 변경사항 기록

각 커밋 후에는 CHANGELOG.md를 업데이트하세요:

1. **Unreleased** 섹션에 변경사항 추가
2. 버전을 릴리스할 때:
   - 날짜 추가
   - **Unreleased** 섹션의 내용을 새 버전으로 이동
   - 버전 태그 생성

### 작성 규칙

- 사용자 중심으로 작성 (기술적 세부사항보다 사용자 관점)
- 명확하고 간결하게 작성
- 관련 이슈 번호 포함 (예: `(#123)`)
- 그룹화: 유형별로 그룹화하여 작성

### 예시

```markdown
### 추가
- 대상자 검색 기능 (#45)
- 대여 기록 내보내기 기능 (CSV, PDF) (#67)

### 수정
- 대상자 수정 시 상태 초기화 오류 수정 (#89)
- 모바일 화면에서 테이블 레이아웃 깨짐 문제 수정 (#92)
```
