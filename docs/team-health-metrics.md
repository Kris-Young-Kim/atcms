# 팀 건강도 지표 가이드

**프로젝트 코드**: ATCMP-2026  
**작성일**: 2025-01-27  
**버전**: 1.0

---

## 📋 개요

이 문서는 AT-CMP 프로젝트 팀의 건강도를 측정하고 추적하기 위한 지표를 정의합니다. 스프린트 완료율, 코드 리뷰 시간, PR 머지 시간 등을 통해 팀의 효율성과 협업 상태를 평가합니다.

---

## 🎯 목적

1. **팀 효율성 측정**: 정량적 지표로 팀 효율성 평가
2. **협업 상태 파악**: 코드 리뷰, PR 머지 시간 등으로 협업 상태 파악
3. **블로커 관리**: 블로커 해결 시간 추적을 통한 효율성 개선
4. **프로세스 개선**: 팀 건강도 지표 분석을 통한 프로세스 개선

---

## 📊 주요 지표

### 1. 스프린트 완료율 (Sprint Completion Rate)

**정의**: 계획된 스토리 포인트 대비 완료된 스토리 포인트 비율

**계산 방법**:
```
스프린트 완료율 = 완료된 스토리 포인트 / 계획된 스토리 포인트 × 100%
```

**측정 주기**: 스프린트 종료 시

**목표**: 80% 이상

**등급**:
- **90% 이상**: 매우 우수
- **80-90%**: 우수
- **70-80%**: 보통
- **70% 미만**: 개선 필요

**평가 항목**:
- 계획된 작업 완료율
- 스프린트 목표 달성 여부
- 미완료 작업 및 이유

---

### 2. 코드 리뷰 시간 (Code Review Time)

**정의**: PR 생성부터 승인까지 소요된 시간

**측정 항목**:
- **PR 생성 시간**: PR 생성 일시
- **첫 리뷰 시간**: 첫 리뷰 코멘트 시간
- **승인 시간**: PR 승인 시간
- **리뷰 시간**: 첫 리뷰 ~ 승인 시간

**목표**:
- **첫 리뷰 시간**: < 4시간
- **리뷰 시간**: 평균 < 24시간

**등급**:
- **목표 달성**: 우수
- **목표의 1.5배 이내**: 보통
- **목표의 1.5배 초과**: 개선 필요

---

### 3. PR 머지 시간 (PR Merge Time)

**정의**: PR 생성부터 머지까지 소요된 시간

**측정 항목**:
- **PR 생성 시간**: PR 생성 일시
- **머지 시간**: PR 머지 일시
- **전체 시간**: PR 생성 ~ 머지 시간

**목표**: 평균 < 48시간

**등급**:
- **< 24시간**: 매우 우수
- **24-48시간**: 우수
- **48-72시간**: 보통
- **> 72시간**: 개선 필요

---

### 4. 블로커 해결 시간 (Blocker Resolution Time)

**정의**: 블로커 발생부터 해결까지 소요된 시간

**측정 항목**:
- **블로커 발생 시간**: 블로커 이슈 생성 시간
- **블로커 해결 시간**: 블로커 이슈 종료 시간
- **해결 시간**: 발생 ~ 해결 시간

**목표**: 평균 < 4시간

**블로커 유형**:
- **기술적 블로커**: 기술적 문제로 인한 블로커
- **의존성 블로커**: 외부 의존성 문제로 인한 블로커
- **리소스 블로커**: 리소스 부족으로 인한 블로커

**등급**:
- **< 2시간**: 매우 우수
- **2-4시간**: 우수
- **4-8시간**: 보통
- **> 8시간**: 개선 필요

## 📈 스프린트 완료율 추적

### 데이터 수집

**수집 시점**: 스프린트 종료 시

**수집 데이터**:
- 계획된 스토리 포인트
- 완료된 스토리 포인트
- 미완료 스토리 포인트
- 미완료 이유

**저장 위치**: GitHub Projects 또는 Notion

---

### 완료율 분석

#### 완료율 트렌드

- **증가 추세**: 계획 정확도 향상, 팀 효율성 개선
- **감소 추세**: 계획이 비현실적이거나 팀 리소스 부족

#### 미완료 원인 분석

- **스코프 크림**: 작업 범위 확장
- **예상보다 복잡**: 작업 복잡도 과소 평가
- **블로커**: 블로커로 인한 지연
- **리소스 부족**: 인력 또는 시간 부족

## 🔍 코드 리뷰 시간 추적

### 측정 방법

1. **GitHub API 활용**
   - PR 생성 시간 추출
   - 리뷰 코멘트 시간 추출
   - PR 승인 시간 추출

2. **데이터 분석**
   - 리뷰 시간 계산
   - 평균 리뷰 시간 계산
   - 리뷰 시간 트렌드 분석

---

### 리뷰 시간 분석

#### 리뷰 시간별 분포

- **< 4시간**: 즉시 리뷰
- **4-12시간**: 당일 리뷰
- **12-24시간**: 하루 내 리뷰
- **> 24시간**: 지연 리뷰

#### 리뷰 시간 개선 방안

1. **리뷰 정책 수립**
   - 리뷰 시간 목표 설정
   - 리뷰 우선순위 체계

2. **리뷰 자동화**
   - 자동 리뷰 도구 활용
   - 리뷰 요청 알림 설정

3. **리뷰 문화 개선**
   - 빠른 리뷰 장려
   - 리뷰 시간 목표 공유

### 측정 방법

1. **GitHub API 활용**
   - PR 생성 시간 추출
   - PR 머지 시간 추출
   - 머지 시간 계산

2. **데이터 분석**
   - 평균 머지 시간 계산
   - 머지 시간 트렌드 분석
   - 머지 시간별 분포

---

### 머지 시간 분석

#### 머지 시간별 분포

- **< 24시간**: 매우 빠른 머지
- **24-48시간**: 빠른 머지
- **48-72시간**: 보통 머지
- **> 72시간**: 느린 머지

#### 머지 시간 개선 방안

1. **작은 PR 장려**
   - 큰 PR을 작은 PR로 분할
   - PR 크기 가이드라인

2. **자동화**
   - CI/CD 자동화
   - 자동 머지 규칙 설정

3. **프로세스 개선**
   - 머지 프로세스 간소화
   - 머지 권한 명확화
