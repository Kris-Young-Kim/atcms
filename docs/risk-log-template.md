### 위험 카테고리 설명

**기술 위험**:
- 기술 스택 선택 문제
- 기술적 난이도 예상 초과
- 외부 라이브러리 호환성 문제

**일정 위험**:
- 작업 시간 예상 초과
- 마일스톤 달성 어려움
- 스프린트 지연

**리소스 위험**:
- 팀원 이탈
- 예산 부족
- 인프라 용량 부족

**외부 의존성 위험**:
- 외부 API 장애
- 외부 서비스 지연
- 협력사 일정 지연

**기타 위험**:
- 법적/규제 준수 문제
- 보안 취약점
- 데이터 손실 위험

### 위험 점수별 대응 전략

**P0 (위험 점수 9)**:
- 즉시 완화 전략 수립 및 실행
- 일일 모니터링
- PM 및 개발 리더 즉시 알림

**P1 (위험 점수 6)**:
- 우선 완화 전략 수립
- 주 2회 모니터링
- 다음 스프린트에 완화 조치 포함

**P2 (위험 점수 3-4)**:
- 계획적 완화 전략 수립
- 주 1회 모니터링
- 다음 스프린트 또는 분기 계획에 포함

**P3 (위험 점수 1-2)**:
- 정기 검토 (월 1회)
- 필요시 완화 전략 수립
- 장기 계획에 포함
