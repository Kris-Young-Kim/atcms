# Notion 프로젝트 공간 생성 가이드

**프로젝트 코드**: ATCMP-2026  
**작성일**: 2025-01-27  
**버전**: 1.0

---

## 📋 개요

이 문서는 AT-CMP 프로젝트의 Notion 작업 공간을 생성하고 구성하는 방법을 안내합니다. Notion을 사용하여 프로젝트 문서, 회의록, 작업 관리 등을 통합적으로 관리할 수 있습니다.

---

## 🎯 목표

1. 프로젝트 전용 Notion 워크스페이스 생성
2. 프로젝트 대시보드 구성
3. 문서 통합 관리 체계 구축
4. 팀 협업 페이지 설정

---

## 📝 Step 1: Notion 워크스페이스 생성

### 1.1 새 워크스페이스 생성 (선택사항)

**방법 1: 새 워크스페이스 생성**
1. [Notion](https://www.notion.so) 접속
2. 좌측 사이드바에서 **+ Add a workspace** 클릭
3. 워크스페이스 이름 입력: `AT-CMP Project`
4. 워크스페이스 타입 선택: **Company** 또는 **Personal**
5. **Continue** 클릭

**방법 2: 기존 워크스페이스 활용**
- 기존 워크스페이스에 새 프로젝트 페이지를 생성하는 방식 권장

### 1.2 프로젝트 페이지 생성

1. 워크스페이스 사이드바에서 **+ Add a page** 클릭
2. 페이지 이름 입력: `AT-CMP (ATCMP-2026)`
3. 페이지 아이콘 설정 (선택사항):
   - 프로젝트 로고 또는 이모지 사용
4. **Create** 클릭

---

## 📊 Step 2: 프로젝트 대시보드 구성

### 2.1 대시보드 템플릿 사용

1. 생성된 프로젝트 페이지에서 **/template** 입력
2. **Project Dashboard** 템플릿 선택 또는 커스텀 구성

### 2.2 권장 페이지 구조

다음과 같은 구조로 프로젝트 페이지를 구성합니다:

```
AT-CMP (ATCMP-2026)
├── 📋 프로젝트 개요
│   ├── 프로젝트 정보
│   ├── 팀 구성원
│   ├── 목표 및 KPI
│   └── 마일스톤
├── 📚 문서
│   ├── 기술 문서
│   ├── 회의록
│   ├── 가이드 문서
│   └── 아키텍처 문서
├── 📝 작업 관리
│   ├── Sprint 계획
│   ├── 이슈 추적
│   └── 액션 아이템
├── 📊 리포팅
│   ├── 일일 리포트
│   ├── 주간 리포트
│   └── 스프린트 리뷰
└── 🔗 링크
    ├── GitHub 저장소
    ├── 배포 환경
    └── 모니터링 대시보드
```

## 📄 Step 3: 주요 페이지 생성

### 3.1 프로젝트 개요 페이지

**내용**:
- 프로젝트명: AT-CMP (Assistive Technology Case Management Platform)
- 프로젝트 코드: ATCMP-2026
- 프로젝트 기간: 2025년 11월 ~ 2026년 8월 (9개월)
- 현재 상태: 개발 단계
- 팀 구성원 목록
- 주요 목표 및 KPI
- 마일스톤 일정

**생성 방법**:
1. 프로젝트 페이지에서 **+ Add a page** 클릭
2. 페이지 이름: `프로젝트 개요`
3. 내용 작성

### 3.2 문서 페이지

**내용**:
- 기술 문서 링크
- 회의록 아카이브
- 가이드 문서 목록
- 아키텍처 문서

**생성 방법**:
1. 프로젝트 페이지에서 **+ Add a page** 클릭
2. 페이지 이름: `문서`
3. 하위 페이지 생성:
   - `회의록`
   - `기술 문서`
   - `가이드 문서`

### 3.3 작업 관리 페이지

**내용**:
- Sprint 계획 및 진행 상황
- GitHub Issues 연동 (선택사항)
- 액션 아이템 추적

**생성 방법**:
1. 프로젝트 페이지에서 **+ Add a page** 클릭
2. 페이지 이름: `작업 관리`
3. **Database** → **Table** 또는 **Board** 선택
4. 데이터베이스 구성:
   - **Title**: 작업 제목
   - **Status**: To Do, In Progress, Done
   - **Assignee**: 담당자
   - **Priority**: P0, P1, P2, P3, P4
   - **Due Date**: 마감일
   - **Sprint**: 스프린트 번호

### 3.4 리포팅 페이지

**내용**:
- 일일 리포트 템플릿
- 주간 리포트 템플릿
- 스프린트 리뷰 템플릿

**생성 방법**:
1. 프로젝트 페이지에서 **+ Add a page** 클릭
2. 페이지 이름: `리포팅`
3. 하위 페이지 생성:
   - `일일 리포트`
   - `주간 리포트`
   - `스프린트 리뷰`

## 🔗 Step 4: 외부 도구 연동

### 4.1 GitHub 연동

**연동 방법**:
1. GitHub 저장소 페이지에서 **Settings** → **Integrations** → **Notion** 선택
2. Notion에서 **Settings & Members** → **Connections** → **GitHub** 선택
3. 권한 부여 후 연동 완료

**연동 효과**:
- GitHub Issues를 Notion 데이터베이스로 가져오기
- PR 상태를 Notion에서 확인
- 커밋 히스토리 연동

### 4.2 Slack 연동

**연동 방법**:
1. Notion에서 **Settings & Members** → **Connections** → **Slack** 선택
2. Slack 워크스페이스 선택 및 권한 부여
3. 알림 채널 설정

**연동 효과**:
- Notion 페이지 업데이트 시 Slack 알림
- 댓글 및 작업 업데이트 알림

---

## 👥 Step 5: 팀 권한 설정

### 5.1 팀원 초대

1. 프로젝트 페이지에서 **Share** 버튼 클릭
2. **Invite people** 선택
3. 팀원 이메일 주소 입력
4. 권한 설정:
   - **Full access**: PM, 개발 리더
   - **Can edit**: 개발자, 디자이너
   - **Can view**: 외부 협력자 (필요시)

### 5.2 권한 매트릭스

| 역할 | 권한 | 설명 |
|------|------|------|
| PM | Full access | 모든 페이지 생성/수정/삭제 가능 |
| 개발 리더 | Full access | 모든 페이지 생성/수정/삭제 가능 |
| 개발자 | Can edit | 페이지 수정 가능, 삭제 불가 |
| 디자이너 | Can edit | 페이지 수정 가능, 삭제 불가 |
| 외부 협력자 | Can view | 페이지 조회만 가능 |

## 📋 Step 6: 템플릿 생성

### 6.1 회의록 템플릿

**생성 방법**:
1. `문서` → `회의록` 페이지에서 **+ New** 클릭
2. 템플릿으로 저장:
   - **Templates** → **New template** 선택
   - 템플릿 이름: `회의록 템플릿`
   - 표준 회의록 형식으로 구성

**템플릿 내용**:
- 회의 정보 (날짜, 시간, 참석자)
- 안건
- 논의 내용
- 결정 사항
- 액션 아이템

### 6.2 리포트 템플릿

**일일 리포트 템플릿**:
- 오늘 완료한 작업
- 내일 계획된 작업
- 블로커 및 이슈

**주간 리포트 템플릿**:
- 주간 목표 달성도
- 완료된 작업 목록
- 진행 중인 작업
- 다음 주 계획

**스프린트 리뷰 템플릿**:
- 스프린트 목표 달성도
- 완료된 작업
- 미완료 작업 및 이유
- 데모 항목
- 회고

## 📚 사용 가이드

### 개발자 워크플로우

1. **일일 작업 시작**
   - 일일 리포트 페이지에서 오늘 작업 계획 작성
   - 작업 관리 데이터베이스에서 할당된 작업 확인

2. **회의 참석**
   - 회의록 페이지에서 회의록 작성 또는 확인
   - 액션 아이템을 작업 관리에 추가

3. **문서 작성**
   - 기술 문서 페이지에 새 문서 작성
   - GitHub 문서와 연동하여 일관성 유지

### PM 워크플로우

1. **주간 계획**
   - 주간 리포트 페이지에서 주간 목표 설정
   - Sprint 계획 페이지에서 스프린트 계획 수립

2. **진행 상황 추적**
   - 작업 관리 데이터베이스에서 진행 상황 확인
   - 블로커 및 이슈 파악

3. **리포팅**
   - 주간 리포트 작성
   - 스프린트 리뷰 작성
