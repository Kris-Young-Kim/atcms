# 위험 관리 가이드

**프로젝트 코드**: ATCMP-2026  
**작성일**: 2025-01-27  
**버전**: 1.0

---

## 📋 개요

이 문서는 AT-CMP 프로젝트에서 위험을 체계적으로 관리하기 위한 가이드입니다. 위험 식별, 평가, 완화, 모니터링의 전체 프로세스를 다룹니다.

---

## 🎯 목적

1. **위험 조기 발견**: 위험을 조기에 발견하고 대응
2. **위험 관리 체계화**: 일관된 방식으로 위험 관리
3. **프로젝트 성공 확률 향상**: 위험 관리로 프로젝트 성공 확률 향상
4. **이해관계자 신뢰도 향상**: 체계적인 위험 관리로 신뢰도 향상

---

## 🔄 위험 관리 프로세스

### 단계 1: 위험 식별

#### 위험 식별 방법

1. **브레인스토밍**
   - 팀 회의에서 위험 브레인스토밍
   - 과거 프로젝트 경험 기반
   - 체크리스트 기반 식별

2. **정기 검토**
   - 주간 위험 검토 회의
   - 스프린트 계획 시 위험 검토
   - 마일스톤 리뷰 시 위험 검토

3. **이슈 분석**
   - 발생한 이슈로부터 위험 도출
   - 블로커 분석으로 위험 식별
   - 변경 요청 분석으로 위험 식별

#### 위험 카테고리

- **기술 위험**: 기술 스택, 아키텍처, 라이브러리 관련
- **일정 위험**: 작업 시간, 마일스톤, 릴리즈 일정 관련
- **리소스 위험**: 인력, 예산, 인프라 관련
- **외부 의존성 위험**: 외부 API, 서비스, 협력사 관련
- **기타 위험**: 법적, 보안, 데이터 관련

---

### 단계 2: 위험 평가

#### 영향도 평가

[위험 평가 매트릭스](./risk-assessment-matrix.md) 참조

- 영향도 척도: 1-5점
- 기술적 영향
- 일정 영향
- 비용 영향
- 품질 영향

#### 확률 평가

[위험 평가 매트릭스](./risk-assessment-matrix.md) 참조

- 확률 척도: 1-5점
- 과거 데이터 기반
- 현재 상황 분석

#### 위험 점수 계산

**위험 점수** = 영향도 점수 × 확률 점수

#### 위험 등급 결정

[위험 평가 매트릭스](./risk-assessment-matrix.md) 참조

- P0 (긴급): 점수 10-25
- P1 (우선): 점수 6-9
- P2 (모니터링): 점수 3-5
- P3 (정기 검토): 점수 1-2

---

### 단계 3: 완화 전략 수립

#### 전략 선택

[위험 완화 전략](./risk-mitigation-strategies.md) 참조

- 회피 (Avoidance)
- 전이 (Transfer)
- 완화 (Mitigation)
- 수용 (Acceptance)

#### 실행 계획 수립

- 완화 조치 정의
- 담당자 지정
- 일정 설정
- 비용 추정

---

### 단계 4: 완화 전략 실행

#### 실행 프로세스

1. **완화 조치 시작**
   - 담당자에게 작업 할당
   - 일정 설정 및 추적

2. **진행 상황 모니터링**
   - 정기적 진행 상황 확인
   - 이슈 발생 시 즉시 대응

3. **완화 효과 평가**
   - 영향도 재평가
   - 확률 재평가
   - 위험 점수 재계산

---

### 단계 5: 위험 모니터링

#### 모니터링 주기

- **P0**: 일일 모니터링
- **P1**: 주 2회 모니터링
- **P2**: 주 1회 모니터링
- **P3**: 월 1회 모니터링

#### 모니터링 항목

- 위험 상태 변화
- 완화 전략 진행 상황
- 영향도 및 확률 변화
- 새로운 위험 발생 여부

---

### 단계 6: 위험 종료

#### 종료 조건

- 위험이 완전히 제거됨
- 위험이 완화되어 수용 가능한 수준
- 위험이 발생하여 해결됨
- 위험이 더 이상 관련 없음

#### 종료 프로세스

1. **종료 검토**
   - 위험 상태 확인
   - 완화 효과 평가

2. **문서화**
   - 위험 관리 과정 문서화
   - 교훈 및 개선 사항 기록

3. **이슈 종료**
   - 위험 로그 상태 업데이트
   - 관련 이슈 종료

---

## 👥 역할 및 책임

### PM (Project Manager)

- 위험 관리 프로세스 주도
- 위험 검토 회의 주재
- 위험 완화 전략 승인
- 이해관계자 커뮤니케이션

---

### 기술 리더

- 기술 위험 평가
- 기술 위험 완화 전략 수립
- 기술 위험 완화 실행

---

### 개발자

- 위험 식별 및 보고
- 위험 평가 지원
- 완화 전략 실행

---

### 팀 전체

- 위험 식별에 적극 참여
- 위험 발생 시 즉시 보고
- 완화 전략 실행 지원

---

## 📅 주기적 위험 검토

### 주간 위험 검토 회의

**일정**: 매주 금요일 오후 4:00 (30분)

**안건**:
1. 진행 중인 위험 상태 확인
2. 새로운 위험 식별
3. 위험 등급 재평가
4. 완화 전략 진행 상황 확인

**참석자**: PM, 기술 리더, 팀 전체

**회의록**: [위험 검토 템플릿](./templates/risk-review-template.md) 사용

---

### 월간 위험 리포트

**일정**: 매월 마지막 주

**내용**:
1. 위험 현황 요약
2. 새로 발견된 위험
3. 완화된 위험
4. 위험 트렌드 분석

**형식**: [위험 리포트 템플릿](./templates/risk-report-template.md) 사용

## 📊 위험 추적

### 위험 로그

- **저장 위치**: GitHub Issues 또는 Notion
- **템플릿**: [위험 로그 템플릿](./risk-log-template.md)
- **업데이트 주기**: 위험 상태 변경 시 즉시 업데이트

### 위험 대시보드

- **위험 현황**: 위험 등급별 분포
- **완화 진행률**: 완화 중인 위험 비율
- **위험 트렌드**: 시간별 위험 변화

---

## 🚨 위험 알림 체계

### 알림 기준

- **P0 위험 발견**: 즉시 PM, 기술 리더 알림
- **새로운 위험 발견**: 당일 내 팀 알림
- **위험 등급 변경**: 관련 담당자 알림
- **완화 전략 지연**: 담당자 및 PM 알림

### 알림 방법

- **긴급**: Slack, 이메일, 전화
- **일반**: Slack, 이메일
- **정보**: Slack, 주간 리포트

## 📝 위험 관리 체크리스트

### 위험 발견 시

- [ ] 위험 로그 항목 생성
- [ ] 영향도 및 확률 평가 완료
- [ ] 위험 점수 계산 완료
- [ ] 위험 등급 결정 완료
- [ ] 완화 전략 수립 완료
- [ ] 관련 이슈 생성 (필요시)

- [ ] 완화 조치 시작
- [ ] 정기적 상태 업데이트
- [ ] 영향도 및 확률 재평가 (월 1회)
- [ ] 완화 전략 효과 검증

### 위험 완화 완료 시

- [ ] 위험 상태를 "완화됨" 또는 "종료됨"으로 변경
- [ ] 완화 과정 문서화
- [ ] 교훈 및 개선 사항 기록
- [ ] 관련 이슈 종료 (필요시)
