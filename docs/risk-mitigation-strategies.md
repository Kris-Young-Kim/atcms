# 위험 완화 전략 가이드

**프로젝트 코드**: ATCMP-2026  
**작성일**: 2025-01-27  
**버전**: 1.0

---

## 📋 개요

이 문서는 프로젝트에서 발생할 수 있는 위험에 대한 완화 전략을 정의합니다. 위험 등급에 따라 적절한 완화 전략을 선택하고 실행 계획을 수립합니다.

---

## 🎯 목적

1. **체계적인 위험 대응**: 위험 유형에 따라 적절한 전략 선택
2. **완화 전략 표준화**: 일관된 방식으로 위험 완화
3. **효과적인 위험 관리**: 위험 완화 효과 극대화
4. **학습 및 개선**: 위험 완화 경험을 통한 프로세스 개선

---

## 🛡️ 완화 전략 유형

### 1. 회피 (Avoidance)

**정의**: 위험의 원인을 제거하거나 위험을 피하는 전략

**적용 시점**: 위험의 영향도가 매우 높고 회피가 가능한 경우

**방법**:
- 위험 요인 제거
- 다른 접근 방식 선택
- 위험 발생 가능성 제거

**예시**:
- 불안정한 외부 API 사용 중단
- 위험한 기술 스택 선택 회피
- 복잡한 기능 단순화

**장점**:
- 위험 완전 제거
- 추가 비용 없음

**단점**:
- 프로젝트 범위 변경 필요
- 기회 비용 발생 가능

---

### 2. 전이 (Transfer)

**정의**: 위험을 다른 조직이나 파트너로 이전하는 전략

**적용 시점**: 위험을 다른 조직이 더 잘 관리할 수 있는 경우

**방법**:
- 외부 서비스 사용 (SaaS)
- 보험 가입
- 외부 전문가 고용
- 책임 전가 계약

**예시**:
- 클라우드 서비스 사용으로 인프라 위험 전이
- 외부 개발팀에 일부 기능 아웃소싱
- 보안 감사 전문업체 고용

**장점**:
- 위험 관리 전문성 활용
- 내부 리소스 절약

**단점**:
- 비용 증가
- 제어권 감소

---

### 3. 완화 (Mitigation)

**정의**: 위험의 영향도나 확률을 줄이는 전략

**적용 시점**: 위험을 완전히 제거할 수 없지만 줄일 수 있는 경우

**방법**:
- 예방 조치 실행
- 모니터링 강화
- 대체 계획 수립
- 추가 리소스 투입

**예시**:
- 외부 API 장애 대비 폴백 메커니즘 구축
- 코드 리뷰 강화로 버그 감소
- 백업 계획 수립
- 추가 테스트 수행

**장점**:
- 위험 영향 감소
- 프로젝트 범위 유지

**단점**:
- 추가 비용 및 시간 필요
- 완전한 제거 불가능

---

### 4. 수용 (Acceptance)

**정의**: 위험을 수용하고 발생 시 대응하는 전략

**적용 시점**: 위험의 영향도가 낮거나 완화 비용이 위험보다 큰 경우

**방법**:
- 위험 모니터링
- 대응 계획 수립
- 비상 계획 준비

**예시**:
- 낮은 확률의 기술적 위험 수용
- 예상치 못한 상황 대비 비상 계획
- 위험 발생 시 즉시 대응 체계 구축

**장점**:
- 비용 절감
- 유연한 대응

**단점**:
- 위험 발생 시 대응 필요
- 예상치 못한 비용 발생 가능

---

## 📊 위험 등급별 완화 전략

### P0 (위험 점수 10-25): 긴급 대응

**전략**: 회피 또는 즉시 완화

**조치**:
1. **즉시 조치** (24시간 이내)
   - 위험 원인 분석
   - 긴급 대응 계획 수립
   - 담당자 지정 및 실행

2. **단기 조치** (1주일 이내)
   - 완화 전략 실행
   - 모니터링 체계 구축
   - 일일 진행 상황 확인

3. **중기 조치** (1개월 이내)
   - 완화 전략 효과 평가
   - 필요시 전략 조정
   - 재발 방지 조치

**모니터링**: 일일

---

### P1 (위험 점수 6-9): 우선 대응

**전략**: 완화 또는 전이

**조치**:
1. **단기 조치** (1주일 이내)
   - 완화 전략 수립
   - 실행 계획 작성
   - 담당자 지정

2. **중기 조치** (1개월 이내)
   - 완화 전략 실행
   - 모니터링 체계 구축
   - 주간 진행 상황 확인

3. **장기 조치** (3개월 이내)
   - 완화 전략 효과 평가
   - 필요시 전략 조정

**모니터링**: 주 2회

---

### P2 (위험 점수 3-5): 모니터링 후 대응

**전략**: 완화 또는 수용

**조치**:
1. **중기 조치** (1개월 이내)
   - 완화 전략 수립
   - 필요시 실행 계획 작성

2. **장기 조치** (3개월 이내)
   - 필요시 완화 전략 실행
   - 모니터링 체계 구축

**모니터링**: 주 1회

---

### P3 (위험 점수 1-2): 정기 검토

**전략**: 수용

**조치**:
1. **장기 조치** (3개월 이내)
   - 모니터링 체계 구축
   - 필요시 대응 계획 수립

**모니터링**: 월 1회

## 🔍 위험 유형별 완화 전략

### 기술 위험

**회피**:
- 불안정한 기술 스택 회피
- 검증된 기술 선택

**완화**:
- 기술 검증 프로세스 수립
- PoC (Proof of Concept) 수행
- 전문가 자문

**전이**:
- 외부 기술 컨설팅
- 기술 파트너십

---

### 일정 위험

**회피**:
- 비현실적인 일정 설정 회피
- 버퍼 시간 확보

**완화**:
- 작업 우선순위 조정
- 추가 리소스 투입
- 스코프 조정

**전이**:
- 외부 개발팀 아웃소싱
- 프리랜서 고용

---

### 리소스 위험

**회피**:
- 핵심 인력 이탈 방지
- 계약 조건 개선

**완화**:
- 지식 공유 및 문서화
- 백업 인력 확보
- 교육 및 온보딩

**전이**:
- 외부 인력 채용
- 파트너십 구축

---

### 외부 의존성 위험

**회피**:
- 외부 의존성 최소화
- 자체 솔루션 구축

**완화**:
- 폴백 메커니즘 구축
- 다중 공급자 확보
- 모니터링 강화

**전이**:
- SLA 계약
- 보험 가입
